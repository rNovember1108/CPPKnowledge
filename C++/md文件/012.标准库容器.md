# 标准库容器

## 引言

容器（Container）是一种特殊的数据结构，用来组织和存储数据。C++标准库提供了多种类型的容器，每种容器都有其特定的数据组织方式和操作方式。以下是C++中最常用的容器：

1. **`vector`**：动态数组，可以在尾部进行高效的插入和删除操作，也可以随机访问

2. **`list`**：双向链表，可以在任何位置进行高效的插入和删除操作。

3. **`deque`**：双端队列，可以在头部和尾部进行高效的插入和删除操作。

4. **`set/multiset`**：基于红黑树的有序集合。`set`中的元素按照特定的顺序排列，且每个元素只出现一次。`multiset`则允许元素重复。

5. **`map/multimap`**：基于红黑树的有序键值对，`map`中的每个键只出现一次，`multimap`则允许键重复。

6. **`unordered set/unordered multiset`**：基于哈希表的无序集合。`unordered set`中的每个元素只出现一次，`unordered multiset`则允许元素重复。

7. **`unordered map/unordered multimap`**：基于哈希表的无序键值对，`unordered map中的每个键只出现一次，`unordered multimap`则允许键重复。

8. **`stack`**：栈，后进先出（LIFO）的数据结构。

9. **`queue`**：队列，先进先出（FIFO）的数据结构。

10. **`priority queue`**：优先队列，内部元素按照优先级进行排序。

## 顺序容器和关联容器

**顺序容器（Sequence Containers）**

- `std::vector`：动态数组，支持随机访问和动态大小调整。

- `std::deque`：双端队列，支持在两端进行快速插入和删除。

- `std::list`：双向链表，支持任意位置的快速插入和删除。

- `std::forward_list`：单向链表，支持任意位置的快速插入和删除，但没有双向迭代器。

- `std::array`：静态数组，固定大小，支持随机访问。

- `std::string`：字符串类，封装了对字符串的操作。

**关联容器（Associative Containers）**

-  `std::set`：有序集合，存储唯一的元素。

- `std::multiset`：有序集合，允许存储重复的元素。

- `std::map`：有序映射，存储键值对，每个键唯一。

- `std::multimap`：有序映射，允许存储重复的键值对。

- `std::unordered_set`：无序集合，存储唯一的元素。

- `std::unordered_multiset`：无序集合，允许存储重复的元素。

- `std::unordered_map`：无序映射，存储键值对，每个键唯一。

- `std::unordered_multimap`：无序映射，允许存储重复的键值对。

除了上述容器，在 C++11 中还引入了一些辅助容器类模板，如`std::queue`、`std::stack`、`std::priority_queue`，它们是基于顺序容器和适配器设计的。

**顺序容器适用于需要按照元素插入的顺序进行访问的情况，而关联容器适用于需要根据键值进行查找和排序的情况。**

## 容器迭代器失效

常见容器的迭代器失效情况：

`std::vector`：

- 在插入或删除元素时，会导致所有迭代器失效。

- 在在容器的中间位置插入或删除元素时，后续的迭代器也会失效。

- 在对容器进行重新分配内存时，迭代器也会失效。

`std::deque`：

- 在在容器的中间位置插入或删除元素时，会导致所有迭代器失效。

- 在对容器进行重新分配内存时，迭代器也会失效。

`std::list`：

- 在插入或删除元素时，不会导致迭代器失效（除了指向要删除的元素的迭代器）。

- 对其他元素进行插入或删除操作不会使迭代器失效。

`std::map `和` std::set`：

- 在插入元素时，不会导致迭代器失效。

- 在删除元素时，被删除元素的迭代器失效，其他迭代器仍然有效。

`std::unordered_map` 和 `std::unordered_set`：

- 在插入元素时，可能会导致迭代器失效。

- 在删除元素时，被删除元素的迭代器失效，其他迭代器仍然有效。

需要注意的是，这些规则只是通常情况下的迭代器失效情况，并不是绝对的规则。

## vector和list区别

**内部实现**：

- `vector`是一个动态数组，它的元素在内存中是连续存放的。

- `list`则是一个双向链表，它的元素在内存中可能分布在各个不同的地方，元素之间通过指针进行连接。

**访问方式**：

- `vector`支持随机访问，可以直接通过下标访问元素，访问速度快。

- `list`只支持顺序访问，如果要访问某个元素，需要从头（或尾）开始逐个遍历，直到找到该元素，因此访问速度较慢。

**插入和删除**：

- `vector`在尾部插入和删除元素非常高效，但在中间或头部插入和删除元素则需要移动大量元素，效率较低。

- `list`在任何位置插入和删除元素都非常高效，因为只需要修改相应的指针即可。

**空间效率**：

- `vector`由于是连续存储，空间利用率高，但需要预留额外空间以支持动态扩展。

- `list`由于使用了指针，每个元素都需要额外的空间存放前后元素的地址，因此空间利用率较低。

## map和unordered_map区别

`map`和`unordered_map`都是C++标准库中的关联容器，它们都能存储键值对的元素，但是它们的内部实现和性能特性有所不同。

- `map`：内部使用红黑树实现，元素会根据键值自动排序。查找、插入和删除操作的时间复杂度为`O(log n)`。`map`在遍历时，元素的顺序是升序排列的。

- `unordered_map`：内部使用哈希表实现，元素的排列顺序是不确定的，不会自动排序。理想情况下，查找、插入和删除操作的时间复杂度为`O(1)`。但在哈希冲突严重的情况下，可能会退化到`O(n)`。`unordered_map`在遍历时，元素的顺序是不确定的。

## Vector细则

**`reserve`和`resize`区别**

- `reserve`是用来预分配vector的内存空间。它会改变`vector`的`capacity`（容量），但是不会改变`vector`的`size`（大小）。也就是说，调用`reserve`不会增加`vector`中元素的数量，只是预先分配了内存，以便在将来添加新元素时能够更快。
- `resize`则是改变`vector`的`size`（大小）。如果新的大小大于当前的大小，那么会添加新的元素到`vector`中，这些新元素的值将会被初始化为默认值。如果新的大小小于当前的大小，那么将会删除多余的元素。在这个过程中，`vector`的`capacity`可能会改变，如果新的大小大于当前的`capacity`，那么会自动增加`capacity`来容纳新的元素。

以下是`reserve`和`resize`的一个例子

```cpp
std::vector<int> vec;
std::cout << vec.size() << ", " << vec.capacity() << std::endl; // 输出：0, 0

vec.reserve(5);
std::cout << vec.size() << ", " << vec.capacity() << std::endl; // 输出：0, 5

vec.resize(5);
std::cout << vec.size() << ", " << vec.capacity() << std::endl; // 输出：5, 5
```

在这个例子中，可以看到，调用`reserve`预分配了内存，但是并没有增加元素的数量，而调用`resize`则增加了元素的数量。

**扩容机制**

`std::vector` 是 C++ STL 中一种动态数组，其大小可以在运行时改变。当我们向 `std::vector` 添加元素时，如果当前分配的内存不足以存放新的元素，`std::vector` 就需要进行扩容。

`std::vector` 的扩容机制如下：

1. 当 `std::vector` 的容量不足以存放新的元素时，它会申请一块新的内存空间。这块新的内存空间的大小通常是当前容量的两倍。这种策略被称为倍增策略

2. `std::vector` 会将旧的元素复制到新的内存空间中。

3. 释放旧的内存空间。

这种扩容机制可以保证 `std::vector` 的插入操作的平摊时间复杂度为 O(1)。但是，由于需要复制元素和释放旧的内存空间，扩容操作的实际时间复杂度为 O(n)，其中 n 是 `std::vector` 中的元素数量。

`std::vector` 的扩容策略通常是**倍增**，也就是每次扩容时，新的内存空间的大小是当前容量的两倍。如果当前 `std::vector` 的容量是 n，那么下一次扩容时，它会申请大小为 2n 的内存空间。但是这种策略并非所有编译器和标准库都会使用，有的可能会选择在每次扩容时增加固定数量的内存。

例如，如果你在初始状态下向一个空的 `std::vector` 添加元素，它可能首先申请大小为 1 的内存空间。当你再次添加元素时，它会发现当前的内存空间不足，然后申请大小为 2 的内存空间。再次添加元素时，它可能会申请大小为 4 的内存空间，依此类推。

**注意，不同的编译器和标准库可能会使用不同的扩容策略。上面描述的是一种常见的策略，但并非所有实现都会这样做。例如，一些实现可能会选择在每次扩容时增加固定数量的内存，而不是倍增。**

**另外，为了避免频繁的扩容操作，如果你已经知道 `std::vector` 最终需要存储的元素数量，可以使用 `std::vector::reserve` 方法预先分配足够的内存。**

**push_back和emplace_back的区别**

`push_back`和`emplace_back`都是用于向vector的末尾添加元素的函数，但它们的工作方式略有不同。

- `push_back`: 这个函数首先创建一个新的元素，然后将这个元素复制或移动到vector的末尾。这意味着如果你正在添加的元素是一个复杂的对象，那么`push_back`会创建这个对象的一个副本，这可能会涉及到一些额外的开销。

- `emplace_back`: 这个函数直接在vector的末尾创建一个新的元素，而不需要先创建一个临时对象，然后再将它复制或移动到vector。这就避免了不必要的复制或移动操作，从而提高了性能。

因此，在添加复杂对象到vector时，`emplace_back`通常比`push_back`更高效。然而，对于简单的数据类型（例如int、float等），两者的性能差异通常可以忽略不计。
