# 进程与线程

## 概念

**线程是CPU调度的基本单位，进程是资源分配的基本单位**‌，两者在操作系统管理中形成包含关系并具有显著差异。

‌**基本定义**‌

- ‌**进程**‌：操作系统进行资源分配的最小单位，代表正在执行的程序实例。每个进程拥有独立的虚拟地址空间、内存资源和文件句柄等系统资源。‌

- ‌**线程**‌：操作系统调度的最小单位，是进程内的执行单元。同一进程的多个线程共享进程资源，但具备独立的执行栈和程序计数器。‌

‌**核心关系**‌

- ‌**包含关系**‌：一个进程至少包含一个主线程，线程是进程的执行路径。‌

- ‌**资源共享**‌：线程共享进程的地址空间、文件描述符等资源，但拥有独立的线程本地存储和寄存器状态。‌

- **生命周期**‌：进程终止时所有线程随之消亡，线程崩溃可能导致整个进程终止。‌‌

**主要区别**

- ‌**资源分配**‌：进程拥有独立内存空间，线程仅共享进程资源；‌

- ‌**切换开销**‌：进程切换需更新页表、寄存器等，开销大（约比线程切换高10-100倍），线程切换仅需保存少量寄存器状态；‌

- ‌**通信机制**‌：进程间需通过IPC（如管道、消息队列）通信，线程间可直接读写共享内存（需同步机制）；‌

- ‌**容错性**‌：进程崩溃不影响其他进程，线程崩溃可能导致整个进程终止。‌‌

## 进程创建

**在C++中创建进程**

在C++中创建进程通常涉及到操作系统提供的API。在不同的操作系统中，创建进程的API有所不同。下面将分别介绍在Windows和类Unix（如Linux和macOS）系统中如何创建进程。

- 在Windows系统中，你可以使用`CreateProcess`函数来创建一个新的进程。

- 在类Unix系统中，你可以使用`fork`和`exec`系列函数来创建一个新的进程。

**在Qt中创建进程**

在Qt中，你可以使用`QProcess`类相关的系列函数来创建一个新的进程。

## 线程创建

**在C++中创建线程**

C++11引入了`std::thread`类简化线程创建，可以分别使用函数指针、Lambda表达式和成员函数都行。

```
#include <iostream>
#include <thread>

void thread_function() 
{
    std::cout << "Thread function executing" << std::endl;
}

int main() 
{
    // 创建并启动线程
    std::thread t(thread_function);

    // 等待线程结束
    t.join();
    return 0;
}
```

线程管理

- `join()`：等待线程结束；

- `detach()`：分离线程（后台运行）；

- `joinable()`：检查线程是否可`join`。

注意事项

- ‌**资源管理**‌：C++的`std::thread`使用RAII，析构时若未`join/detach`会`terminate`；

- **参数生命周期**‌：确保线程参数在线程使用期间有效；

- ‌**线程安全**‌：共享数据需要同步机制（`mutex`等）；

**在Qt中创建线程**

1. 子线程继承`QThread`，然后重写`run()`函数来实现线程逻辑，并在外层调用`start()`启动执行。

2. 使用`moveToThread()`，任务类`Task`需要继承`QObject`，然后写一个执行任务的函数，创建一个`QThread`线程，然后调用`moveToThread()`将对象移动到创建的子线程对象中，然后启动执行。

3. 使用`QtConcurrent`，`QtConcurrent` 是 Qt 提供的一个高级并发编程模块，属于 `QtConcurrent` 命名空间，旨在简化多线程任务的执行。它支持并行执行算法（如 `map`、`filter`、`reduce`），还支持异步任务运行和结果管理，不需要显式管理 `QThread`，以下着重介绍此方法。

## QtConcurrent

**QtConcurrent::run()**

每调用一次QtConcurrent::run()函数，就新建立一个线程。

![](https://i-blog.csdnimg.cn/direct/e2d8369607cf47b584dd6412b09dfa15.png)

异步运行一个函数（无返回值）

```
QtConcurrent::run([]() 
{
    qDebug() << "后台线程中执行任务：" << QThread::currentThread();
});
```

异步运行一个带参数的函数（有返回值）

```
int add(int a, int b) 
{
    return a + b;
}

QFuture<int> future = QtConcurrent::run(add, 3, 5);
future.waitForFinished();

// 后续获取结果
int result = future.result();  // 阻塞直到完成
qDebug() << "结果是：" << result;
```

绑定类成员函数

```
class Worker 
{
public:
    int multiply(int x, int y) 
    {
        return x * y;
    }
};

int main()
{
    Worker worker;
    QFuture<int> future = QtConcurrent::run(&worker, &Worker::multiply, 4, 6);
    int result = future.result();  // 阻塞直到完成获取结果
    qDebug() << "结果是：" << result;
}

或者

{
public:
    static int multiply(int x, int y)
```

**QtConcurrent::map()**

并发修改容器元素（原地修改）

```
QList<int> numbers = {1, 2, 3, 4, 5};

auto doubleIt = [](int &n) {
    n *= 2;
};

QFuture<void> future = QtConcurrent::map(numbers, doubleIt);
future.waitForFinished();

//numbers将变为: 2 4 6 8 10
```

**QtConcurrent::maped()**

并发映射容器到新容器

```
int square(int n) 
{
    return n * n;
};

QList<int> numbers = {1, 2, 3};
QList<int> squares = QtConcurrent::mapped(numbers, square).results();

//squares将输出: 1 4 9
```

**QtConcurrent::filter()**

并行过滤 

```
QList<int> list = { 1, 2, 3, 4, 5 };

// 原地过滤
QFuture<void> future = QtConcurrent::filter(list, [](int x) {
   return x % 2 == 0; // 只保留偶数
});
future.waitForFinished();

// list 现在是 {2, 4}
```

**QtConcurrent::filtered()**

并行过滤并返回新容器

```
bool filter(int n)
{
    return n % 2 == 0;
};

QList<int> list = { 1, 2, 3, 4, 5 };
QFuture<int> future = QtConcurrent::filtered(list, filter);
QList<int> result = future.results();

//squares将输出: 2 4
```

**QtConcurrent::filteredReduced()**

并行过滤然后归约并返回新容器

```
bool filter(int n)
{
    return n % 2 == 0;
};

void sum(int& result, int x)
{
    result + = x;
};

QList<int> list = { 1, 2, 3, 4, 5 };
int result = QtConcurrent::filteredReduced(list, filter, sum);

//result 将输出:6
```
