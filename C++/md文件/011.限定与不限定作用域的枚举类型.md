# 限定与不限定作用域的枚举类型

## 限定作用域的枚举型别的优势

### 对初始化变量名称的影响

对于不限定作用域的枚举类型，其枚举量作用域的范围是包含这个枚举型别的作用域。

    enum clolr { 
        blue, red 
    }; 
    auto red = 0; // 错误，red已经在范围内被声明过了 

限定作用域的枚举型别则不一样 

    enum class clolr { 
        blue, red 
    }; 
    auto red = false; 
    color a = color::red; 

对于限定作用于的枚举型别，可以降低对名字空间的污染。

### 避免隐式类型转换

    enum clolr { 
        blue, red 
    }; 
    color c = red; 
    if(c < 5.5){ 
        ... 
    } 

在上述例子中，`c`被隐式转换为了浮点型变量

而在限定作用域的枚举类型中，上述隐式转换的问题则不复存在

    enum class clolr { 
        blue, red 
    }; 
    color c = color::red; 
    if(c < 5.5){ // 错误，color型别无法与double型进行比较 
        ... 
    } 

### 降低因改动产生的编译成本

由于编译器会针对用户定义`enum`进行优化，因此，对于以下形式的枚举型别，编译器可能会选择`char`作为其底层型别而不是int。

    enum color{red, blue};

然而，若将`color`的定义换为以下形式，`char`变量不足以表示其底层变量，则可能需要重新编译所有系统：

    enum color{ 
        red = 0xfffffff, 
        blue = 10000 
    } 

限定作用域的枚举型别支持前置声明，可以提前指定底层型别的类型，前置声明必须指定枚举成员类型，隐式指定或显式指定均可。

    //错误，不限定作用域枚举类型的前置声明应指定成员类型
    //(VS2022并没有报错，gcc version 6.3.0报错,建议都显示指定)
    enum etype1;   
    
    enum etype2:unsigned int;      //正确
    enum class etype3;          //正确，限定作用域枚举类型成员有默认类型int 
    enum class etype4:unsigned int;  //正确，前置声明指定枚举成员类型

与所有的声明语句一样，枚举的声明和定义中描述的成员类型必须一致，另外不能在同一个上下文中先声明一个不限定作用域的枚举类型再声明一个同名的限定作用域枚举类型。

    enmu class Clolor:char; //前置声明枚举类
    void Foo(Color*p); //前置声明的使用
    //....................
    enum class Color:char{RED,GREEN,BLACK,WHITE}; //前置声明的定义

## 不限定作用域的枚举型别的优势

因为枚举成员可以自动转换成整数类型，在需要传入整数的程序中可以直接写枚举成员，可读性高，得益于不限定作用域枚举型别允许隐式类型转换的特性，使用以下方法可以减少开发的负担：

    // 三个成员分别代表name，email，声望值
    using userInfo = tuple<string, string, int>;  
    userInfo a; 
    auto val = get<1>(a); // 得到了域1的值，即email的值 
    
    using userInfo = tuple<string, string, int>; 
    enum widget{uName, uEmail, uReputation}; 
    userInfo a; 
    auto val = get<uEmail>(a); // 也得到了域1的值

与此同时，使用限定作用域的枚举型别，实现上述功能则会复杂很多：

    enum class widget{uName, uEmail, uReputation}; 
    userInfo a; 
    auto val = get<static_cast<size_t>(widget::uEmail)>(a);
