## Qt插件加载

将插件的工作目录path通过 `qputenv`设置到环境变量中

```
qputenv("PATH", QString("%1;%2")
.arg(QString::fromLocal8Bit(qgetenv("PATH")), path).toLocal8Bit());
```

- 以前一直使用的`[static] bool QDir::setCurrent(const QString &path)`，先设置插件工作路径，加载成功后再恢复程序工作路径，此方法不是线程安全的，多线程下多个插件加载会导致工作路径错乱。
- `qputenv` 设置的环境变量仅在当前程序运行时有效，程序结束后就会被清除。
- 环境变量在不同的操作系统和环境中可能会有所不同，需要注意跨平台的兼容性。

## Qt数据库释放

在Qt框架中，`QSqlDatabase`的线程安全规则要求数据库连接的关闭和释放必须与创建连接在同一线程中完成，这是其核心设计约束。具体表现为：

### 线程绑定机制

- ‌**连接生命周期管理**‌：每个`QSqlDatabase`实例及其关联的`QSqlQuery`对象必须严格限定在创建它们的线程内使用，跨线程操作会导致未定义行为或程序崩溃。例如在线程A创建的连接，在线程B中调用`close()`或`removeDatabase()`将触发异常。

- ‌**连接名复用限制**‌：虽然不同线程可通过唯一连接名访问同一物理数据库，但每个线程需独立创建自己的`QSqlDatabase`实例。共享连接名时，必须确保每个线程实例的打开/关闭操作完全隔离。

### 释放流程

1. **关闭连接顺序**‌：必须先关闭所有关联查询(`QSqlQuery`)，再关闭数据库连接，最后调用`removeDatabase()`。逆向操作会触发"connection is still in use"警告。典型代码结构如下：

```
// 在当前线程内执行释放
QString connName = db.connectionName();
db.close();
QSqlDatabase::removeDatabase(connName);  // 必须在创建连接的线程调用
```

2. ‌**资源清理时机**‌：`removeDatabase()`调用后需确保事件循环执行完成，立即重建连接可能导致资源未完全释放。建议通过QTimer延迟重建或显式调用`QCoreApplication::processEvents()`。
